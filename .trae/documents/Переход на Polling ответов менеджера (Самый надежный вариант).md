# План перехода на Long Polling для ответов менеджера

## 1. Отказ от Webhooks и ngrok
Убираем нестабильную часть с `aiohttp` и `pyngrok`. Бот будет работать проще и надежнее.

## 2. Реализация проверки ответов (Polling)
Добавляем в `scheduler` (планировщик) новую задачу, которая запускается каждые 10-15 секунд.
*   **Задача:** `check_manager_replies`
*   **Логика:**
    1.  Берем список активных сделок из базы данных (у кого есть `lead_id`).
    2.  Для каждой сделки запрашиваем список событий/примечаний через API (`GET /api/v4/leads/{id}/notes`).
    3.  Ищем примечания, созданные за последние 15 секунд.
    4.  Если текст начинается с `>>>`, отправляем в Telegram.
    5.  (Опционально) Помечаем примечание как "отправленное" (чтобы не дублировать), например, меняем его текст или храним ID отправленных примечаний в памяти.

## 3. Обновление `kommo.py`
Добавляем метод `get_lead_notes(lead_id)`, который возвращает список примечаний.

**Почему это лучше:**
*   Не зависит от настроек вебхуков amoCRM.
*   Работает локально без внешних IP.
*   Легче отлаживать.
